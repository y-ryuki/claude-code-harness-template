# /plan: 実装計画立案

タスク: $ARGUMENTS

## やること

1. **ゴール確認**: 何を達成すれば完了か（成功基準を明文化）
2. **コンテキスト把握**: `knowledge-explorer` で関連ファイル・既存パターンを調査
3. **タスク分解**: ゴールを 3〜7 のステップに分解
4. **影響範囲**: 変更が必要なファイル一覧
5. **リスク**: 失敗時の影響、考えられる落とし穴
6. **検証方法**: テスト戦略、確認手順

## 出力フォーマット

```markdown
## 📋 実装計画: [タスク名]

### ゴール
- [ ] 成功基準1
- [ ] 成功基準2

### コンテキスト
- 関連ファイル:
  - `src/...`
- 既存パターン:
  - ...

### ステップ
1. [ ] **Step 1**: 説明
   - 影響ファイル: `src/...`
2. [ ] **Step 2**: ...
3. [ ] **Step 3**: ...

### リスク
- ⚠️ ...

### 検証方法
- [ ] ユニットテスト追加: `tests/...`
- [ ] 手動確認: ...

### 見積もり
- 工数: 〇時間
```

## ルール

1. ステップは **30 分以内** で完了するサイズに分解
2. 各ステップに **影響ファイル** を明示
3. 大きすぎる場合は「これ自体が複数 PR に分割可能」と提案
4. ユーザーが GO したら実装を開始する
